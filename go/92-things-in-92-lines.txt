tip 41/92 (from various sources) Use `RWMutex` Wisely. A `sync.RWMutex` is not a universally faster `sync.Mutex`. It has higher overhead for coordination. tip 42/92 (from various sources) Prevent Goroutine Leaks with `select`. A goroutine blocked on sending or receiving from a channel will leak if the other end is gone. tip 87/92 (from Evan Wallace, skew) Build Custom Tools if Standard Ones Slow You Down. Evan built the Skew language specifically because he wanted an optimizing compiler that targeted JS/C#/C++ natively with fast compile times. If the ecosystem tools force you into a sluggish implementation process, taking a detour to build your own toolchain is actually the fastest path to shipping.
tip 88/92 (from Evan Wallace, esbuild) Single-Maintainer Means Singular Focus. Esbuild is famously maintained almost entirely by one person. This allows the entire architecture to exist in one brain, ensuring every optimization aligns. To survive the implementation phase of a hyper-complex system, limit the number of cooks in the kitchen so architectural integrity remains unbroken.
tip 89/92 (from Evan Wallace, esbuild) Parse it Once, Parse it Fast. Most JS toolchains pipe strings between 5 different generic AST tools. Esbuild survives by parsing, linking, and printing in a highly integrated pipeline. Avoid toolchain chaining. An integrated pipeline built from scratch will always obliterate a pipeline glued together from generic parts.
tip 90/92 (from Evan Wallace, esbuild) Fail Fast and Loud on Unhandled Edge Cases. Esbuild's codebase makes heavy use of `panic` in default switch cases for enums. "This way developers adding a new record type will quickly find where they need to add code." Crashing the compiler immediately is the safest way to survive adding new features to a massive codebase.
tip 91/92 (from Evan Wallace, Figma) Performance is a Feature, Not an Optimization. "Performance and quality are some of our most important features... you only notice them when they aren't there, but they make all the difference." Treat latency targets as strict binary constraints from day one, not as technical debt to be solved later. If it's slow, it's broken.
tip 92/92 (from Evan Wallace, esbuild) Control the Output Exactly. Esbuild goes as far as implementing its own JS string minifier, detecting when `\n` can be shorter than `\n`, or when small strings should be inlined. True survival in systems programming requires controlling every byte of the final payload, trusting nothing to default stringification.
